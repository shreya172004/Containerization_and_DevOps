System Architecture

Two containers communicate over an IPvlan network with statically assigned IP addresses. The backend connects to the database directly via the IPvlan static IP, and data is persisted using a named Docker volume.

Client (Browser / Postman)
↓   HTTP Requests   ↓

IP: 172.22.208.11  |  Port: 8000
↓   PostgreSQL Connection via IPvlan   ↓
postgres_db — PostgreSQL 15 Alpine
IP: 172.22.208.10  |  Port: 5432

Setup & Execution

01
Verify Docker Installation
Confirm Docker and Docker Compose are installed.
$ docker --version
Docker version 29.2.1, build a5c7197
$ docker compose version
Docker Compose version v5.0.2
02
Create IPvlan Network
Manually create the external IPvlan network before running Docker Compose.
$ docker network create -d ipvlan \
  --subnet=172.22.208.0/24 \
  --gateway=172.22.208.1 \
  -o parent=eth0 mynetwork
3c13de8da92470216f345f5516e48ce390...
03
Build and Start Containers
Docker Compose builds both images and starts the services.
$ docker compose up --build
[+] Building 14.2s (4/4) FINISHED
✔ Container postgres_db   Healthy
✔ Container backend_api   Created
04
Verify Static IP Addresses
Confirm both containers received their IPvlan static IPs.
$ docker inspect backend_api | grep IPAddress
"IPAddress": "172.22.208.11"
$ docker inspect postgres_db | grep IPAddress
"IPAddress": "172.22.208.10"
05
Test API Endpoints
Health check, insert record, and fetch records — all working.
# Health Check
{"status":"healthy","service":"fastapi-backend"}

Volume Persistence Test
